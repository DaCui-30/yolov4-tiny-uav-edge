# YOLOv4-Tiny FPGA Accelerator for UAV Edge Deployment

FPGA-based hardware accelerator for YOLOv4-tiny object detection, optimized for resource-constrained UAV edge applications.

## Hardware Platform

- **Current Platform**: PYNQ-Z2 (Zynq-7020) — for proof-of-concept and validation
- **Tool**: Vivado HLS 2022.1 + Vitis 2022.1

> **Note**: The PYNQ-Z2 is a development board. For actual UAV deployment, the design can be ported to smaller FPGA chips (e.g., Zynq UltraScale+ ZU3EG) with custom PCB design.

## Project Structure

```
yolov4-tiny-uav-edge/
├── hls/top/src/     # HLS C++ IP cores
│   ├── conv.cpp     # Convolution with double-buffered memory hierarchy
│   ├── sparse_conv.cpp # Sparse convolution (exploits ReLU sparsity)
│   ├── maxpool.cpp  # Max pooling
│   ├── upsample.cpp # Upsampling
│   └── std_conv.cpp # Standard convolution
├── Vitis/src/       # ARM-side application (C++)
│   ├── main.cpp     # Main application
│   ├── motion_adaptive.cpp # Motion-adaptive inference (UAV power saving)
│   ├── basic_block.h
│   ├── nms.cpp/h   # Non-maximum suppression
│   └── sd_io.cpp/h # SD card I/O
├── tools/predictor/ # Python inference verification
│   ├── yolo.py     # YOLO model
│   ├── bn_merge.py  # Batch normalization fusion
│   └── predict.py   # Prediction script
└── folded_weights/   # BN-fused weights (generated by tools/predictor)
```

## Key Optimizations

### 1. Double-Buffered Memory Hierarchy
Overlaps data movement and computation using ping-pong buffers, reducing idle cycles.

### 2. Sparse Convolution
Exploits ReLU-induced sparsity to skip zero-weight computations, reducing MAC operations.

### 3. Motion-Adaptive Inference
Detects UAV motion between frames; skips inference when hovering (power saving).

## Workflow

### 1. Train YOLOv4-Tiny (Python)
```bash
cd tools/predictor
python bn_merge.py --weights yolov4_tiny_weights_coco.pth
python predict.py --image test.jpg
```

### 2. Generate FPGA Weights (BN Fusion)
```bash
python bn_merge.py  # Folds BN into conv weights
# Output: folded_weights/ directory
```

### 3. Synthesize HLS IP Core
```bash
cd hls/top
vivado_hls -f run_hls.tcl
# Generates RTL (Verilog/VHDL)
```

### 4. Deploy on PYNQ-Z2
```bash
cd Vitis
vitis -workspace workspace
# Load bitstream, run inference
```
